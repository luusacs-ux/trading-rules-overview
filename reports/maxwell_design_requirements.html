Maxwell — Design & Requirements Document

Conversational Options Analysis Engine • Last updated: 2026-05-24

1. Overview

Maxwell is a conversational financial analyst that lives inside Telegram. Users send natural-language questions about stocks, options, crypto, or the market, and Maxwell responds with quantitative analysis wrapped in a distinctive street-smart personality. The system uses a 2-pass Ollama LLM architecture backed by real-time options math and news sentiment.

Key Design Principles

  • Math first, opinion second — every response is grounded in computed metrics, never pure LLM hallucination
  • Zero API cost for LLM — Ollama runs locally, no OpenAI/Anthropic spend
  • Sub-90s response time — full pipeline (ticker extraction + data fetch + analysis + response generation) within Ollama timeout
  • Personality is a feature — cocky, street-smart tone differentiates from generic bots and keeps the group engaged

4.2 Calculated Metrics

Call/Put Volume Split

call_pct = call_volume / (call_volume + put_volume) * 100
put_pct = put_volume / (call_volume + put_volume) * 100

Calculated Simple percentage split of total options volume between calls and puts for the nearest-Friday expiry.

Put/Call Ratio (Volume)

pcr = put_volume / call_volume

Calculated Directional flow indicator. >1.0 = bearish flow, <0.7 = bullish flow.

Put/Call Ratio (Open Interest)

oi_pcr = put_oi / call_oi

Calculated Positioning indicator (longer-term than volume P/C). ≥1.3 = put-heavy, ≤0.7 = call-heavy.

ATM Implied Volatility

atm_iv = mean(closest_call_iv, closest_put_iv)

Calculated Finds the call and put strikes nearest to current price, averages their implied volatilities. Used as the IV input for all subsequent probability calculations.

IV Rank (0-100%)

rolling_vol = 20-day rolling std(daily returns) * sqrt(252) [annualized realized vol]
iv_rank = (current_atm_iv - 52wk_low_rv) / (52wk_high_rv - 52wk_low_rv) * 100

Calculated Where current ATM IV sits within the 52-week range of 20-day realized volatility. High rank = expensive options. Requires ≥30 trading days of history.

Max Pain

For each possible settlement strike S:
total_pain(S) = sum[ call_OI(K) * max(S - K, 0) for K in all_strikes ]
+ sum[ put_OI(K) * max(K - S, 0) for K in all_strikes ]
max_pain = strike S that minimizes total_pain(S)

Calculated The price at which option holders collectively lose the most money. Theory: market makers push price toward max pain at expiry. Displayed with directional arrow showing distance from current price.

Expected Move

expected_move = price * IV * sqrt(DTE / 365)

Calculated Standard IV-derived 1-standard-deviation expected price range by expiration. Displayed as ±$ and ±%.

Unusual Activity Detection

median_vol = median(all_strike_volumes) [floor: 10]
Flag if: volume > 5x median AND volume ≥ 100 contracts

Calculated Scans every strike in calls and puts. Flags strikes with volume spikes well above the chain's baseline. Reports side (CALL/PUT), strike, volume, multiple-vs-median, and OI. Top 5 by volume returned.

Support & Resistance (3-Method Composite)

Calculated Each price level is scored by three independent methods, then combined:

Method 1: Gamma Exposure (GEX) — weight 40%

gamma(S, K, T, sigma) = N'(d1) / (S * sigma * sqrt(T)) [Black-Scholes gamma]
d1 = (ln(S/K) + (r + 0.5*sigma^2)*T) / (sigma * sqrt(T))

For each strike across up to 6 nearest expirations:
call_gex += gamma * call_OI * 100 * price (positive)
put_gex -= gamma * put_OI * 100 * price (negative)

Strikes with highest absolute GEX = dealer hedging walls

Method 2: OI Walls — weight 40%

oi_by_strike = aggregate(call_OI + put_OI) across up to 6 expirations
Strikes with highest aggregate OI = potential pinning/barrier levels

Method 3: Volume Profile — weight 20%

3-month price history bucketed into 50 equal-width price bins
Each bin weighted by trading volume at that price level
High-volume nodes = historically contested price zones

Composite Score

score = 0.4 * (|GEX| / max_GEX) + 0.4 * (OI / max_OI) + 0.2 * (VP / max_VP)

Strikes are split into above-price (resistance) and below-price (support). Top 3 of each returned, sorted closest-to-price first, ranked by composite score.

Touch Probabilities (per S/R level)

sigma = IV * sqrt(DTE / 365)
z = ln(level / price) / sigma
P(touch) = 2 * (1 - Phi(|z|)) [reflection principle, log-normal GBM]

Calculated Probability of price touching each S/R level by a given date. Computed at two horizons: this Friday and next Friday. Based on the reflection principle for geometric Brownian motion.

Sentiment Score (composite)

Calculated Integer scoring system combining three signals:

SignalConditionPoints
Volume P/C Ratio≥ 1.5-2
≥ 1.0-1
≤ 0.7+1
≤ 0.5+2
OI P/C Ratio≥ 1.3-1
≤ 0.7+1

Result: ≥2 = Bullish  |  ≤-2 = Bearish  |  else = Neutral

News Sentiment Aggregation

Filter articles to relevance > 0.3
avg_score = mean(per_ticker_sentiment_scores)

Labels: ≥0.35 = Bullish | ≥0.15 = Somewhat-Bullish | ≥-0.15 = Neutral
≥-0.35 = Somewhat-Bearish | <-0.35 = Bearish

Calculated from Alpha Vantage raw article scores. Returns label + article count + top 3 bullish/bearish headlines. Cached 12 hours to preserve API quota (25/day free tier).

Signal Sheet

Calculated compute_signals() distills all computed metrics into plain-English signal strings for the LLM. Examples:

  • "Volume P/C ratio is 1.3 — moderately bearish flow"
  • "Max pain $185.00 is 2.1% above — upward magnet"
  • "IV rank 82% — very elevated, premium sellers' market"
  • "Expected move ±$4.50 (±2.3%) by expiry"
  • "Unusual CALL activity at $190.00: 12,500 vol (8.3x median), OI 3,200"

10. Current Limitations & Known Gaps

  • Options data source: yfinance is unofficial and can break on Yahoo API changes. Schwab API integration is planned (slot reserved in options_data.py) for real-time, reliable data.
  • IV Rank uses realized vol as proxy: True IV rank should compare current IV to historical IV, but historical ATM IV isn't available from yfinance. The 20-day realized vol proxy is approximate.
  • Max pain is nearest-expiry only: Doesn't aggregate across expirations. Large OI clusters in later expirations are invisible to this calculation.
  • GEX assumes standard Black-Scholes: No dividend adjustment, no early exercise for American options. Uses risk-free rate of 5% hardcoded.
  • Volume profile is 3-month lookback: May miss longer-term structural S/R from older price action.
  • Crypto: price-only: No options chain analysis for crypto symbols. Only price + news sentiment.
  • Alpha Vantage free tier: 25 requests/day limits how many tickers can get fresh news sentiment. 12h cache helps but can serve stale data.
  • Ollama model dependency: Requires the custom options-analyst Modelfile to be built and loaded. If model is unavailable, falls back to generic llama3.1:8b which lacks financial specialization.
  • No historical tracking: Maxwell doesn't track past predictions or accuracy. No feedback loop on whether his calls were right.
  • Single-expiry focus for main analysis: The primary metrics (volume split, P/C ratio, max pain) use only the nearest Friday expiry. S/R uses multi-expiry but capped at 6.

11. Planned Enhancements

  • Schwab API integration: Real-time, official options data replacing yfinance.
  • Prediction tracking: Log Maxwell's directional calls, compute accuracy over time, display hit rate.
  • Multi-expiry max pain heatmap: Show how max pain shifts across expirations.
  • Dark pool / block trade integration: Unusual activity enriched with institutional flow data.
  • Earnings calendar awareness: Auto-flag upcoming earnings, adjust IV context (pre-earnings crush, etc.).
