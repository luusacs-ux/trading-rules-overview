1. Motivation

The Problem with Hypothesis-First

The existing create_rules_local3.py pipeline tests ~180 hand-coded condition templates (SA01-SA35, LL_lag variants, DV, INV) against historical data. Each condition is a hypothesis: "when RSI ≤ 25 and the candle is green, buy."

Two structural limitations:

  1. Condition space is manually enumerated. Multi-factor market states that don't match templates are invisible.
  2. No multi-timeframe confluence in conditions. Entry signals can't express "15m RSI oversold AND 1h EMA rising AND daily above 50 SMA."

The Outcome-First Alternative

Instead of asking "does this pattern predict a move?", we start from the other end: "before every significant move, what did the market look like?"
  1. Find all historical instances where price moved significantly (the outcomes)
  2. Capture a rich, multi-timeframe snapshot of market state preceding each move
  3. Cluster similar snapshots to discover recurring pre-move structures
  4. Validate each cluster as a predictive pattern against the full dataset
  5. Convert validated patterns into deployable rule files (same format as existing pipeline)

Integration

Produces the exact same output artifacts: rules_{ticker}_{tf}.py, _rule_index_block.py, backtest JSON. Rules flow through existing Phase 6 tier gate, Phases 7-12 sim optimization, Phase 13 deployment — unchanged. Replaces Phases 1-5 only.

Naming: rules use suffix _P (Pattern-discovered): {TICKER}_P_S, {TICKER}_P_L, {TICKER}_P_D, {TICKER}_P_X

2. Pipeline Phases

Phase 0: Data Preparation

Reuses existing download_missing_data(), load_data(), add_indicators(). Downloads OHLCV for all 5 timeframes (1m, 5m, 15m, 1h, daily). No new code needed.

Phase 1: Outcome Labeling

Identify all bars where price made a significant move:

max_advance(i) = max(high[i+1:i+hold]) / close[i] - 1
max_decline(i) = 1 - min(low[i+1:i+hold]) / close[i]

A bar is a bullish outcome if max_advance ≥ threshold before max_decline ≥ stop.

ParameterDefaultRangeDescription
move_threshold_pct2.0%0.5-10.0Min % move to qualify
stop_threshold_pct1.5%0.3-8.0Max adverse move before target
lookforward_bars5 × bars/day1-30 daysForward window
cooldown_barsbars/day-Min spacing between outcomes
min_outcomes3020-100Min outcomes to proceed

Thresholds are asset-adaptive via ATR:

atr_pct = (daily_atr / avg_price) * 100
move_threshold_pct = max(0.5, min(10.0, atr_pct * 1.5))

Outcomes labeled on 5m (finest usable TF with sufficient history). Output: list of (timestamp, direction, actual_move_pct, time_to_target).

Phase 2: Multi-Timeframe Snapshot Capture

For each outcome, look backwards and capture a 210-feature vector. See full feature list below.

Phase 3: Pattern Clustering (HDBSCAN)

Pipeline: Raw features (210 dims) → StandardScaler → PCA (95% variance, ~50-80 dims) → HDBSCAN → Filter → Extract signatures → Convert to conditions

Ensemble: 5 parameter configs, keep clusters appearing in ≥ 3 of 5 runs.

Cluster retained if:

  • ≥ 10 outcome instances
  • Members span ≥ 6 calendar months
  • Mean intra-cluster distance < 2× global mean
  • ≥ 70% same direction (bull or bear)

Phase 4: Pattern Validation

Walk full dataset bar-by-bar, compute snapshot, check distance to each centroid. Grid-search TP/SL/hold.

Out-of-Sample: Train on 2023-01 to 2025-06, validate 2025-07 to present.
Pattern passes if: val WR ≥ 0.8 × train WR, positive PnL, ≥ 3 val trades.
Permutation test: Shuffle labels 100×, pattern significant if p < 0.05.

Phase 5: Rule Generation

Convert cluster's top 4-6 signature features into threshold conditions. Same format as existing SA/LL/DV rules. No numpy/sklearn at runtime.

Phases 6-7: Existing Infrastructure

Hand off to existing tier gate, sim optimization (Phases 6-12), and deploy_worker_rules.py (Phase 13) — unchanged.

4. Clustering Approach

Why HDBSCAN over K-Means?

K-Means requires specifying K upfront. We don't know how many pre-move patterns exist. HDBSCAN discovers the number naturally from data density. It also identifies noise points (expected 30-60% of snapshots) — moves without recurring structure are excluded.

ParameterDefaultRationale
min_cluster_size10Min samples for a pattern
min_samples5Core point density
metriceuclideanStandard for PCA data
cluster_selection_methodeomFavors larger clusters

5. Validation & Overfitting Prevention

5 layers of protection:
  1. Temporal train/validation split — no data leakage
  2. Permutation significance testing — p < 0.05
  3. Minimum trade counts — 10+ for T1, 5+ for T2
  4. 1m simulation optimization — realistic bar-by-bar execution
  5. 3-fold temporal cross-validation — pattern must appear in all folds

9. Risks & Open Questions

Risks:
  1. High feature-to-sample ratio (210 features for ~500 outcomes). PCA mitigates but doesn't eliminate.
  2. Threshold conversion fidelity — continuous centroids to discrete thresholds loses information.
  3. Regime sensitivity — 2023 clusters may not hold in 2026. Temporal spread filter mitigates.

Open Questions:

  1. Should outcomes be labeled on 5m or 15m?
  2. How many signature features per rule? (Default: 4-6)
  3. Should correlation tickers be in clustering or added as supplementary filters in Phase 5?
  4. UMAP vs PCA for dimensionality reduction? (PCA first, UMAP in v2)

class AAPL_P_S5M_00001(BaseAAPL_P_S5m):
    name = "AAPL_P_S5M_00001"

    def evaluate(self, aapl_row):
        rsi = self._safe(aapl_row, "rsi")
        vol_ratio = self._safe(aapl_row, "vol_ratio")
        h1_ema20_slope = self._safe(aapl_row, "h1_ema20_slope")
        m15_bb_position = self._safe(aapl_row, "m15_bb_position")
        daily_price_vs_ema50 = self._safe(aapl_row, "daily_price_vs_ema50")

        if (rsi <= 28 and vol_ratio >= 2.0 and h1_ema20_slope > 0
                and m15_bb_position <= 0.15 and daily_price_vs_ema50 > 0):
            return AaplSignal(
                direction="LONG", tp_pct=2.0, sl_pct=1.5,
                hold_bars=390, cd_bars=78,
                reason="P_S5M_00001: RSI oversold + 1h MA rising + "
                       "vol spike + near 15m lower BB + daily uptrend"
            )
        return None
